# YACS — Yet Another Claude Skills Repo

> **Claude Code用スキル** の厳選されたコレクション。セキュリティ監査からシステム設計、プロセスのゲーミフィケーション、または単に異なった方法で考えることまで、各セッションでできることを拡張します。

---

## これは何ですか？

**YACS** はスキルのリポジトリです — Claude Codeが関連する場合は自動的に呼び出すか、手動で`/skill-name`で呼び出すプロンプトベースのツールです。

一般的なコマンドではありません。各スキルは厳密さと深さをもって特定の種類の問題を解決するために設計されています:

- **セキュリティ監査** — 明らかなものだけでなく、実際の脆弱性を探します
- **品質に関する思考** — 異なった視点で考えさせます
- **データ分析** — 同じ数字の2つの対立した解釈を探索します
- **ゲーミフィケーション設計** — 企業環境に特化しています
- **デザインパターン探索** — あなたの具体的な問題に適用されます

## インストール

### オプション1: npx を使用（推奨）

```bash
npx @mammals-at-work/yacs
```

インストール不要 — 直接実行されます。

### オプション2: グローバルにインストール

```bash
npm install -g @mammals-at-work/yacs
yacs
```

### オプション3: リポジトリから

```bash
git clone https://github.com/munchkin09/YACS.git
cd YACS/cli
npm install
npm start
```

## インストーラーの使い方

インストーラーがあなたをガイドします:

1. **言語選択** — あなたの言語を選びます
2. **インストール位置** — `~/.claude/skills` またはカスタムパスにインストール
3. **スキル選択** — ↑↓で移動、SPACE で選択
4. **確認** — インストール前に選択を確認

**キーボードショートカット:**
- ↑↓ - 移動
- SPACE - 選択/非選択
- a - すべて切り替え
- i - 選択を反転
- Enter - 確認

## カテゴリー別スキル

### 🔒 品質とセキュリティ

#### `/owasp-guardian`
OWASP Top 10の脆弱性についてコードを監査します。正確なスニペット、攻撃ベクトル、修正を提供します。

#### `/llm-safety-checks`
LLM固有の脆弱性についてコード、プロンプト、エージェントフローを監査します: 間接的なプロンプト注入、指示の難読化、ジェイルブレイク、RAG中毒。

#### `/task-flow-planner`
タスクファイル(MDまたはJSON)を読み込み、依存関係とブロッカーを検出し、チーム別に作業を優先順位付けし、テーブル+Mermaidダイアグラムを生成します。

#### `/quality-spark`
ソフトウェア品質についての思考の種を生成します。引数なしで、各呼び出しが異なる視点を提供します。

---

### 💻 開発

#### `/gamify`
企業の内部プロセスのゲーミフィケーション戦略を設計します。動機をマッピングし、メカニクスを選択し、進行状況を設計し、アンチパターンを回避します。

#### `/tech-debt-hunter`
技術負債を特定し、優先順位付けします。実際の影響で分類し、対応しない場合のコストを見積もり、最小限の解決アクションを提案します。

---

### 🏗️ 分析、設計、アーキテクチャ

#### `/adr-writer`
完全で即座にコミットできるアーキテクチャ決定記録(ADR)を生成します。技術決定から、コンテキスト、決定、選択肢、結果を生成します。

#### `/pattern-finder`
具体的な問題に適用可能なデザインパターンを探索します。関係する力を分解し、候補を特定し、トレードオフを比較し、最良の解決策を推奨します。

#### `/architecture-spark`
迅速なアーキテクチャ分析と設計思考。主要な決定を特定し、ソリューションを提案し、リスクをハイライトします。

---

### 🧠 議論とアイデア対立

#### `/red-team`
敵対的な視点からアイデアに異議を唱えます。弱点を特定し、盲点を指摘し、仮定をテストします。

#### `/brainstorm`
構造化されたブレインストーミング。アンカリングバイアスとグループシンク思考を避けるために設計されたプロンプト。視点の変更による新しいアイデアを生成します。

---

### 📊 データと解釈

#### `/data-storyteller`
生のデータを説得力のあるナレーティブに変換します。洞察を抽出し、データの周りに一貫した物語を構築し、重要なことをハイライトします。

#### `/data-debate`
同じデータの2つの対立した解釈を探索します。デフォルトのナレーティブに異議を唱え、隠された視点を発見します。

#### `/deep-research`
トピックの徹底的な探索。ギャップを特定し、仮定に異議を唱え、より深い理解のためのフレームワークを提案します。

#### `/metric-trap`
メトリクスの罠と測定の失敗を特定します。メトリクスがどのように誤解させることができるか、測定していないこと、代替指標を探索します。

## 注記

## 貢献

新しいスキルを追加するには:

1. `skills/<category>/<skill-name>/` にフォルダを作成
2. スキルの説明を含む `SKILL.md` ファイルを含める
3. 必要なスクリプトまたはツールを追加
4. インストーラーが自動的に検出します
